### Screen Reader Support

#### Announcement Pattern

When bookings load:
1. Screen reader announces trip completion: "Upcoming trips loaded, 3 trips available"
2. When user navigates pagination: "Page 2 of 4, showing 3 trips"
3. When carousel moves: "Slide 2 of 5, showing past trip from January 10, 2025"

#### Trip Card Context

Each booking card should announce:
- **Departure & Arrival**: "Flight from New York to London"
- **Date & Time**: "Wednesday, March 12, 8:30 AM"
- **Status**: "Upcoming", "Past", "Confirmed", "Cancelled"
- **Actions Available**: "Check-in available" or "View baggage"

#### Empty State

When no bookings found:
- Screen reader announces: "No upcoming trips found. Add a new trip using the button below."

#### Error State

When API fails:
- Screen reader announces: "Failed to load bookings. Please try again later."
- Retry button is clearly labeled and focusable.

### Best Practices

1. **Semantic Structure**
   - Use `<section>` for major regions (Upcoming, Past)
   - Use `<list>` and `<listitem>` for trip collections
   - Avoid `<div>` spam; leverage native HTML semantics

2. **Focus Management**
   - Ensure all interactive elements are keyboard-accessible
   - Maintain focus order logically (top-to-bottom, left-to-right)
   - Show visible focus indicator (outline, background color)
   - Never use `outline: none` without a custom focus style

3. **Live Regions**
   - Use `ariaLive="polite"` for status messages (loading, pagination changes)
   - Use `ariaLive="assertive"` only for critical alerts (errors, warnings)
   - Avoid excessive updates that cause notification fatigue

4. **Labels & Descriptions**
   - All buttons must have visible or accessible labels
   - Avoid icon-only buttons; include text or proper `ariaLabel`
   - Pagination buttons: "Previous", "Next", "Page X"
   - Action buttons: "Check-in", "Check Baggage", "Manage Booking" (not just icons)

5. **Color & Contrast**
   - Don't rely on color alone to communicate state (upcoming vs. past)
   - Maintain at least 4.5:1 contrast ratio for text
   - Use text labels, badges, or icons in addition to color coding

6. **Touch Targets**
   - Pagination buttons and carousel controls must be at least 44×44 px
   - Ensure sufficient spacing (minimum 8 px) between touch targets
   - Avoid small, cramped clickable areas that frustrate keyboard & mobile users

7. **Error Recovery**
   - Provide clear error messages that describe what went wrong
   - Include a "Retry" button for failed API calls
   - Don't hide booking data permanently; offer workarounds

8. **Animations & Motion**
   - Carousel transitions should respect `prefers-reduced-motion` preference
   - Avoid auto-playing carousels; let users control navigation
   - If animations are present, provide a pause/play control (with proper ARIA)
