# Date Display

<a href="https://www.figma.com/files" target="_blank" rel="noopener noreferrer">Figma design</a> |
<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/date-display" target="_blank" rel="noopener noreferrer">GitHub</a>

`date-display` renders a localized date string by combining `dayjs` formatting with the culture returned by `CultureServiceEx`. Use it whenever you need a plain-text date that adapts to multiple locales without recreating formatting logic inside templates.

## Basic usage

The component receives a `config` object. Provide the date as a native `Date` or a `dayjs` instance and, optionally, the format and culture override.

 > **Tip:** Review the <a href="https://day.js.org/docs/en/display/format" target="_blank" rel="noopener noreferrer">Day.js token reference</a>; the component maps the most common CLDR/Angular tokens to the equivalents documented there.

```html
<date-display [config]="exampleConfig" />
```

```ts
protected readonly exampleConfig = {
	date: dayjs('2015-10-02'),
	format: 'EEE d MMM. y',
};
```

## Accessibility

- The component outputs plain text which inherits surrounding semantics, so place it inside elements that expose the correct role (e.g. `time`, headings, table cells).
- Screen readers pronounce the localized date because the rendered string comes pre-formatted—avoid wrapping the component with additional abbreviations that could conflict.
- Honor the overall document `lang` attribute or set it on the closest ancestor to help pronunciation for mixed-language layouts.
