# Paginator

<a href="https://www.figma.com/design/RQJViheCmmu9SECbNQoQro/AV-%7C-SHIPPED-%7C-Library?node-id=8024-832" target="_blank" rel="noopener noreferrer">Figma Design</a> |
<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/paginator" target="_blank" rel="noopener noreferrer">GitHub</a>

`ds-paginator` renders a navigation bar with previous/next controls and a smart window of page numbers. It clamps invalid inputs, emits the selected page, and adapts to the responsive breakpoint defined by `--paginator-layout-breakpoint`.

## Basic usage

The component consumes a `PaginatorConfig` object. Pass the current page, total page count, and how many items to show at once.

```html
<ds-paginator [config]="resultsPaginator" (selectPageEmitter)="onPage($event)" />
```

```ts
resultsPaginator = {
	currentPage: 5,
	totalPages: 24,
	pagesToShow: 5,
};
```

## Variants

### Window positions

Showcases the paginator when the current page sits near the beginning, middle, or end of the sequence. Designers can confirm when ellipses appear.

<Canvas of={ComponentStories.POSITIONS} />

- **Start**: Highlights the previous button disabled state and hides the leading ellipsis.
- **Middle**: Displays both ellipses, keeping the active page centered.
- **End**: Disables the next button and exposes the jump-to-last button.

### Responsive density

Demonstrates how the component reduces the number of visible page numbers when the viewport matches the responsive breakpoint.

<Canvas of={ComponentStories.RESPONSIVE_DESKTOP} />
<Canvas of={ComponentStories.RESPONSIVE_COMPACT} />

- **Desktop window**: Shows ellipses plus first/last shortcuts for wide layouts.
- **Mobile window**: Shrinks to three numbers with no ellipses for faster tapping.

## Accessibility

- Wraps the list in `<nav role="navigation">` with a translatable `aria-label`.
- Previous/next buttons expose descriptive `aria-label`s and set `disabled` when they cannot be used.
- Active page toggles `aria-current="page"` so screen readers announce the selection.
- Ellipsis buttons are rendered as `aria-hidden` to avoid noisy announcements.
- Responsive state is recalculated upon media-query changes to keep focusable items in range.
