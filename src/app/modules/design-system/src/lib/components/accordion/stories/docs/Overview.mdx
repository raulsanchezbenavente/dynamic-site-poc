# Accordion

[Figma Design](FIGMA_LINK_HERE) |
[GitHub](https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/accordion)

`ds-accordion` is a disclosure component that shows and hides sections of content. It supports two modes: single expand (default) and multiple expand via `allowMultipleExpanded`. Items are always composed using content projection with `ds-accordion-item`.

## Basic usage

The accordion is composed by wrapping `ds-accordion-item` elements inside `ds-accordion`. Each item receives its configuration via direct inputs.

```html
<ds-accordion [config]="accordionConfig">
  <ds-accordion-item title="First section" [isInitiallyExpanded]="true">
    <p>Content for the first section.</p>
  </ds-accordion-item>
  <ds-accordion-item title="Second section">
    <p>Content for the second section.</p>
  </ds-accordion-item>
</ds-accordion>
```

```ts
accordionConfig: AccordionConfig = {
  allowMultipleExpanded: false,
};
```

## Variants

### Behaviors

Use `allowMultipleExpanded: false` (default) when only one section should be visible at a time — for example, FAQ lists or settings panels where context from one section supersedes another. Use `allowMultipleExpanded: true` when users benefit from comparing multiple open sections simultaneously.

<Canvas of={AccordionStories.BEHAVIORS} />

- **Single expand** — opening a new item collapses the previously active one.
- **Multiple expand** — all items can be open at the same time.

### States

Items support four states. Disabled items keep the trigger in the tab order and expose `aria-disabled="true"` to assistive technologies, so users understand why the item is unavailable.

<Canvas of={AccordionStories.STATES} />

- **Active — expanded**: item is interactive and currently open.
- **Active — collapsed**: item is interactive and currently closed.
- **Disabled — collapsed**: item cannot be toggled; content is not accessible.
- **Disabled — expanded**: item was opened on initialisation but cannot be collapsed by the user.

### Content types

Any Angular template can be projected inside a `ds-accordion-item`: plain text, rich HTML, or complex component trees.
