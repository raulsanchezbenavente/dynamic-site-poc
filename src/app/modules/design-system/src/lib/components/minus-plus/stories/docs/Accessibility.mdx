# Minus Plus

## Accessibility

The minus-plus component is a numeric stepper used primarily for passenger counts and quantity selections. It combines two buttons and a read-only input to allow users to increment or decrement a value within a defined range.

### Screen Reader Support

- The value container uses `role="status"`, creating a **live region** that announces value changes automatically.
- Each button has a descriptive `aria-label` that communicates both the action (increase/decrease) and the context (e.g., what is being counted).
- Boundary states are exposed with `aria-disabled="true"` on the corresponding action button.
- The visible input is hidden from assistive technologies via an `aria-hidden="true"` wrapper.

### Best Practices

1. **Use descriptive labels**: Always include the context in `minusLabel` and `plusLabel`. Use "Decrease adults" instead of just "Decrease" so screen reader users understand what they are changing.
2. **Pair with visible context text**: Place a clear visible label near the control (for example, “Adults”), because the input itself is not part of the accessibility tree.
3. **Provide meaningful `statusLabel`**: The `statusLabel` describes the category of the value (e.g., "Adults", "Bags"). Combined with the `role="status"` live region, it ensures value changes are announced contextually.
4. **Don't rely on visual cues alone**: Keep action labels explicit and make boundary states understandable through `aria-disabled` and surrounding context text.
5. **Ensure minimum touch target of 44x44px**: Both the minus and plus buttons should meet WCAG 2.5.5 minimum target size for pointer inputs.
