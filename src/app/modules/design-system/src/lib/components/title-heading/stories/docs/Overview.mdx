# Title Heading

<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/title-heading" target="_blank" rel="noopener noreferrer">GitHub</a>

This guide focuses on how to use `ds-title-heading` in templates: how to configure it, which heading levels and styles to pick, and how to separate semantic HTML from visual appearance.

## Basic usage

The title heading is configured through a `config` object of type `TitleHeadingConfig`. This object controls the heading level, visual style, alignment, intro text, and accessibility visibility.

```html
<ds-title-heading [config]="headingConfig" />
```

```ts
import { TitleHeading, TitleHeadingConfig } from '@dcx/ui/design-system';

const headingConfig: TitleHeadingConfig = {
  title: 'Welcome to our site',
  htmlTag: TitleHeading.H1,
  styleClass: TitleHeading.H1,
  introText: 'A brief introduction paragraph below the heading.',
};
```

## Variants

### Heading levels

Each heading level (H1–H6) renders the appropriate semantic HTML tag with its own visual style.

<Canvas of={TitleHeadingStories.HEADINGS} />

- **H1**: Page title — use once per page for the primary heading.
- **H2**: Section heading — for major content sections.
- **H3**: Subsection heading — within an H2 section.
- **H4–H6**: Deeper nesting levels for complex content hierarchies.

### Semantic vs visual separation

The `htmlTag` and `styleClass` can be set independently. This allows maintaining proper document outline (for accessibility and SEO) while achieving the desired visual design.

<Canvas of={TitleHeadingStories.STYLE_OVERRIDE} />

- Use this when the design requires a visually prominent heading, but the document structure needs a different semantic level.
- Example: An `h2` tag styled as `h1` for visual prominence without breaking the heading hierarchy.

### Alignment

The heading text can be aligned left, center, or right using `horizontalAlignment`.

<Canvas of={TitleHeadingStories.ALIGNMENTS} />

- **Left**: Standard content alignment for LTR layouts.
- **Center**: Centered headings, common for hero sections and landing pages (default).
- **Right**: Right-aligned headings for specific layout needs.

### Visually hidden

When `isVisuallyHidden` is `true`, the heading is hidden from visual display but remains in the DOM for screen readers, maintaining accessibility.

<Canvas of={TitleHeadingStories.VISUALLY_HIDDEN} />

- Use for sections that need a heading for screen reader navigation but should not display one visually.
