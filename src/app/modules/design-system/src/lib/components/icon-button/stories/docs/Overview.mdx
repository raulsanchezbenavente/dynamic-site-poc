This guide focuses on how to use `<icon-button>` in templates: how to configure it, which variants to pick, and critical accessibility requirements for icon-only buttons.

## Basic usage

The icon button is configured through a `config` object. This object controls the icon displayed, visual appearance, size, and critically important ARIA attributes for accessibility.

```html
<icon-button [config]="homeButtonConfig" (clickEvent)="handleHomeClick()" />
```

```ts
homeButtonConfig = {
  icon: {
    name: 'home',
  },
  ariaAttributes: {
    ariaLabel: 'Go to home page', // REQUIRED for accessibility
  },
  layout: {
    style: ButtonStyles.ACTION,
    size: LayoutSize.MEDIUM,
  },
};
```

## Variants

### Sizes

Icon buttons come in three sizes to match different UI contexts.

<Canvas of={IconButtonStories.SIZES} />

- **Small**: Use in compact interfaces, data tables, or as secondary actions in dense layouts
- **Medium** (default): Standard size for most use cases, toolbar actions, and card headers
- **Large**: Use for primary actions or touch-optimized interfaces

### Styles

Icon buttons support four visual styles to establish hierarchy and context.

<Canvas of={IconButtonStories.STYLES} />

- **Action**: High-emphasis buttons for primary icon actions (e.g., submit, favorite, share)
- **Secondary**: Medium-emphasis for secondary actions or grouped toolbar buttons
- **Inverse**: For use on dark backgrounds or colored surfaces
- **Link**: Minimal style that appears like a text link, for tertiary actions

### States

Icon buttons support various interaction states for accessibility and user feedback.

<Canvas of={IconButtonStories.STATES} />

- **Default**: Standard interactive button
- **Disabled**: Removes button from tab order and prevents interaction (`isDisabled: true`)
- **Aria-disabled**: Visually disabled but remains focusable and in tab order (use when user needs context about why it's disabled)
- **Toggle** (`ariaPressed`): For buttons that represent an on/off state (e.g., favorite, mute, bookmark)
- **Expandable** (`ariaExpanded`): For buttons that show/hide content panels or dropdowns
- **Menu** (`ariaHaspopup`): For buttons that open popup menus, dialogs, or other overlays

## Events

The component emits a `clickEvent` when clicked (unless disabled or aria-disabled prevents interaction).

```html
<icon-button
  [config]="deleteConfig"
  (clickEvent)="handleDelete()" />
```

## Critical Accessibility Requirement

**Icon buttons MUST include `ariaLabel` in `ariaAttributes`.** Without it, screen readers announce only "button" with no context about the button's purpose. The component will warn in development mode if `ariaLabel` is missing.

```typescript
// ❌ BAD - No ariaLabel, inaccessible
badConfig = {
  icon: { name: 'trash' },
};

// ✅ GOOD - Clear ariaLabel
goodConfig = {
  icon: { name: 'trash' },
  ariaAttributes: {
    ariaLabel: 'Delete item',
  },
};
```
