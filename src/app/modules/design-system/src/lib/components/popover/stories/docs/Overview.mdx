# Popover

<a href="https://www.figma.com" target="_blank" rel="noopener noreferrer">Figma Design</a> |
<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/popover" target="_blank" rel="noopener noreferrer">GitHub</a>

`ds-popover` reveals rich content when the user clicks or hovers on a trigger. It automatically switches to an off-canvas sheet on compact screens, supports projected headers and body content, and exposes keyboard-friendly behaviors for hover triggers.

## Basic usage

Configure the component with a `PopoverConfig` object and optional `offCanvasConfig`. The trigger/content pair use the dedicated directives.

<Source

## Variants

### Responsive layouts

Demonstrates how the component chooses between the inline popover and the off-canvas drawer depending on the breakpoint.

<Canvas of={ComponentStories.LAYOUTS} />

- **Responsive**: Keeps the inline popover on desktop while switching to the drawer on mobile.
- **Inline only**: Forces the popover layout even on small screens.

### Trigger styles

Showcases the icon trigger and alternative trigger events, including hover interactivity and multiple instances sharing different handlers.

<Canvas of={ComponentStories.TRIGGER_MODES} />

- **Icon trigger**: Uses the default information glyph when no trigger content is projected.
- **Hover trigger**: Opens on pointer hover but still handles keyboard activation and ESC/TAB flows.
- **Hover + handler**: Executes a custom callback before closing to support analytics or chained actions.

### Placement options

Illustrates how `placement` changes the attachment point around the trigger.

<Canvas of={ComponentStories.PLACEMENTS} />

- **Start**: Useful for aligning with labels on the left side of a form.
- **Top**: Keeps the panel above the trigger to avoid covering inputs beneath it.
- **End**: Pins the panel to the right edge for quick-reading flows.

## Accessibility

- Wraps the trigger in a focusable button and restores focus after closing so keyboard users keep context.
- Overrides Bootstrap's default tooltip role to `dialog`, sets `aria-modal`, and wires `aria-controls` to the popover content for screen readers.
- Provides `triggerAriaLabel` for icon triggers and enforces an accessible close button with translated labels.
- Hover triggers automatically convert to click-with-keyboard behavior so users relying on keys can still open the panel.
