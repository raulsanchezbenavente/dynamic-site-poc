# Tag

Tag is the design-system primitive for displaying fare-tier labels (e.g., "Economy S", "Business Flex"). It keeps fare metadata inline with card or itinerary layouts without introducing buttons or interactive affordances. For status messaging or semantic states, use `StatusTagComponent` instead.

## Basic usage

Tags are configured through a `TagConfig` object. Provide the display `text` and the fare-specific `cssClass` so the correct palette token is applied.

```html
<tag [config]="fareTag"></tag>
```

```ts
public fareTag = {
  text: 'Basic',
  cssClass: 'fare1',
};
```

## Appearance

Map the fare priority to the provided `fare1`–`fare4` classes so the tag inherits the correct color token. Keep labels concise—ideally under 20 characters—to avoid wrapping.

### Fare labels

When a fare tier needs a distinct color (as in `summary-cart`), pass the fare priority as part of the `cssClass`. The `SummaryCartDetailComponent` builds it with `cssClass: 'fare' + fareClass`, which maps to the `ds-tag--fareX` tokens defined in `tag.styles.scss`.

Color tokens for each fare tier live in `projects/design-system/assets/tokens/colors/fares.token.scss`; keep any palette updates centralized there so the class names used across summary flows continue to render the correct brand color.

## Recommendations

- Use sentence case labels so multi-word tags remain legible.
- Reserve tags for categorical metadata; use `badge` when emphasis or iconography is needed.
- Combine with tooltips or popovers when the label alone might be ambiguous.
- Use [**Status Tag**](?path=/docs/atoms-badge-tag-status-tag--overview) when communicating a state (success, pending, warning) instead of overloading this fare-specific component.

## Accessibility

- Tags render as plain text. If the label must be read differently for assistive tech, provide that context in the surrounding component (e.g., include the fare type in the card title).
- Maintain sufficient color contrast between the tag background and text when applying custom CSS classes.
- Avoid using tags as interactive controls; use buttons or chips when selection is required.
