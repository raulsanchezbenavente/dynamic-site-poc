# Icon

<a href="FIGMA_LINK_HERE" target="_blank" rel="noopener noreferrer">Figma Design</a> |
<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/icon" target="_blank" rel="noopener noreferrer">GitHub</a>

This guide focuses on how to use `icon` in templates: how to configure it and how to provide accessible labeling when the icon conveys meaning.

The component renders glyphs from the shared UI+ web font, so every icon requested in `config.name` must exist in that font file.

## Basic usage

The icon is configured through a `config` object. This object controls which glyph to display and how screen readers should interpret it.

```html
<icon [config]="iconConfig" />
```

```ts
public iconConfig = {
  name: 'email',
  ariaAttributes: {
    ariaLabel: 'Email notifications',
  },
};
```

## Accessibility

- When `ariaLabel` is provided the component sets `role="img"` and exposes the trimmed label.
- Without a label, the component sets `aria-hidden="true"` so decorative icons are ignored by assistive tech.
- Keep color contrast at least 3:1 against the background. Icons inherit color from surrounding text unless you override via CSS.
- Avoid conveying state solely with color; pair icons with text or tooltips for clarity.
