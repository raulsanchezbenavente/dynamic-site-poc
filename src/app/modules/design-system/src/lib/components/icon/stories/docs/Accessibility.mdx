# Icon

## Accessibility

### ARIA Attributes

Use the icon as either informative (`ariaLabel` provided) or decorative (`ariaLabel` omitted).

#### Required Attributes

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Usage</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><code>ariaLabel</code></td>
      <td>
        Required when the icon communicates meaning on its own. The component trims this value and applies it as <code>aria-label</code>, while also setting <code>role="img"</code>.
      </td>
    </tr>
  </tbody>
</table>

#### Optional State Attributes

<table>
  <thead>

      <th>Attribute</th>

      <td>

      </td>

### Keyboard Navigation

`icon` is a non-interactive element and does not receive keyboard focus.

### Screen Reader Support

- With `ariaLabel`: announced as an image with the provided label.
- Without `ariaLabel`: hidden from the accessibility tree through `aria-hidden="true"`.

### Best Practices

- Provide `ariaLabel` only when the icon adds information not already present in adjacent text.
- Keep decorative icons unlabeled to reduce screen reader noise.
- Do not rely on icon color alone to communicate critical status; pair with visible text.
- Use meaningful, concise labels such as `Error`, `Success`, or `Email notifications`.
