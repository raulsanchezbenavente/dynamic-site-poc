# Data Not Found

<a href="https://www.figma.com/design/RQJViheCmmu9SECbNQoQro/AV-%7C-SHIPPED-%7C-Library?node-id=12577-2894" target="_blank" rel="noopener noreferrer">Figma Design</a> |
<a href="https://github.com/FlyrInc/dcx-ui-av/tree/main/projects/design-system/src/lib/components/data-not-found" target="_blank" rel="noopener noreferrer">GitHub</a>

`data-not-found` communicates empty or unavailable search results. It displays a concise summary, optional secondary guidance, and can host recovery actions through projected content.

## Basic usage

Wrap the component in templates and feed it a `config` object. Project optional actions (links, buttons) to give travelers a next step, but prefer expressing copy/icon/status inside the config so the spacing, typography, and responsive behavior remain consistent.

```html
<data-not-found [config]="emptyResultsConfig">
  <a href="#">Start a new search</a>
</data-not-found>
```

```ts
emptyResultsConfig: DataNotFoundConfig = {
  text: 'We could not find matching trips.',
  secondaryText: 'Try adjusting the travel dates or passenger mix.',
  icon: {
    name: 'airplane-cancel',
    ariaAttributes: { ariaLabel: 'Empty results' },
  },
  statusText: 'Showing alternative suggestions soon',
};
```

## Variants

### Copy hierarchy

Balance `text` and `secondaryText` to explain context and offer guidance.

<Canvas of={ComponentStories.TEXT_STATES} />

- `text` alone keeps the component compact for inline use.
- Adding `secondaryText` helps customer-service flows where troubleshooting steps are needed.
- Use projected content for links/buttons instead of embedding HTML into the `text` copy.

### Content projection

Place inline actions, secondary links, or other markup via `ng-content`.

<Canvas of={ComponentStories.CONTENT_PROJECTION} />

- Links should describe the action (e.g., “Modify search”).
- Keep projected content short so layouts stay balanced on small breakpoints.

## Accessibility

- The wrapper uses semantic text blocks; add `statusText` when asynchronous updates occur so screen readers receive announcements.
- Icons are purely decorative unless you provide `ariaAttributes.ariaLabel`; do so anytime the glyph communicates meaning.
- Ensure projected actions are standard links or buttons so they inherit keyboard and focus behavior automatically.
