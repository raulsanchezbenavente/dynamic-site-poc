`alert-panel` surfaces high-priority information inline with surrounding content. Configure the component through a single `config` object to pick the right alert type, layout, accessibility labels, and optional body copy or projected content.

## Basic usage

The component is configured via a `Partial<AlertPanelConfig>`. Titles and descriptions accept HTML strings, while projected content can introduce buttons or links.

```html
<alert-panel [config]="exampleConfig"></alert-panel>
```

```ts
exampleConfig: AlertPanelConfig = {
	title: 'Check-In Completed',
	description: 'You can now download your <a href="#">boarding pass</a>.',
	alertType: AlertPanelType.SUCCESS,
};
```

## Variants

### Status types

Alert types communicate semantic meaning. Use `WARNING` for actions required, `ERROR` for blocking issues, `SUCCESS` for completion, `INFO` for neutral updates, and `NEUTRAL` for neutral or inactive states.

<Canvas of={ComponentStories.STATUSES} />

- The last example shows `AlertPanelType.NEUTRAL` which provides a muted, inactive appearance.
- You can project links or any rich content within the alert to drive inline actions.

### Content variants

The alert panel supports flexible content configurations: title-only, description-only, or both with projected content.
